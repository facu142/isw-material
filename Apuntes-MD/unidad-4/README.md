## Glosario rápido

- **PPQA**: *Process and Product Quality Assurance* — disciplina de soporte que asegura que tanto el proceso como el producto cumplan con los estándares definidos.
- **TDD**: *Test Driven Development* — escribir el test antes que el código.
- **ATDD**: *Acceptance Test Driven Development* — definir criterios de aceptación con el cliente/PO antes de implementar.
- **SCM**: *Software Configuration Management* — administración de configuración.
- **GAC**: *Grupo de Aseguramiento de Calidad* — equipo independiente que verifica el cumplimiento de estándares.
- **RNF**: *Requerimiento No Funcional* — define **cómo** debe comportarse el sistema, no **qué** hace.
- **CMMI**: *Capability Maturity Model Integration* — modelo de madurez de procesos.
- **SCAMPI**: método de evaluación de CMMI.
